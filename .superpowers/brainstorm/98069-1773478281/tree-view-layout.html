A: Sidebar Tree + Content Table

Persistent tree sidebar for navigation. Content area shows items in selected folder as a table with all metadata columns. Classic file manager feel (VS Code, Finder sidebar).

Pros

  • Full hierarchy always visible
  • Quick navigation between folders
  • Rich metadata in table columns

Cons

  • Takes horizontal space
  • Tree can get deep/cluttered

B: Inline Expandable Tree Table

Single unified table where folders are expandable rows with indentation. Everything in one view - folders and items share the same columns. Like a file explorer in tree mode (Windows Explorer details view).

Pros

  • No wasted horizontal space
  • Everything in one unified view
  • Metadata visible for all items at once
  • Consistent with each other (Notes = Files)

Cons

  • Deep nesting stretches the Name column
  • Need to expand folders one by one

C: Hybrid (Mini Tree + Content Table)

Compact tree sidebar for folder navigation only (no items), content area shows current folder items in table with metadata. Subfolders inside content can also be expanded inline. Best of both worlds.

Pros

  • Quick folder jumping via sidebar
  • Content table has full metadata
  • Sidebar stays compact (folders only)
  • Can collapse sidebar on mobile

Cons

  • Slightly more complex implementation

Shared Design Principle

Whichever layout you pick, both Notes and Files will use the exact same component structure - just with different metadata columns (Notes: tags, content size, attachments; Files: mime type, file size, tags).
